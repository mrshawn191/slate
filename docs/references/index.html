    Edit on Github

    To propose changes to the Developer Portal click the link below and follow these steps:

    1. Sign in to Github. You will have to create an account if you don't already have one.
    2. Click the Fork this repository and propose changes button.
    3. Make your proposed changes to the file. Content is formatted with Markdown.
    4. At the bottom of the page, optionally add a title and description to your changes, and click the Propose file change button.
    5. On the next page, click Create pull request, and then click it again.
    6. You're done! Someone will review your changes and they will appear on the site shortly.

    References

    In-depth information on our payment gateway.

    Payment APIs

    The Payment APIs allows you to tokenize cards, take payments, store customer data and query transactions.

    Onboarding API

    The Onboarding API allows you to submit applications on behalf of sub-merchants.

    Custom Checkout Reference

    Custom Checkout allows you to create a fully customizable payment form while minimizing the scope of your PCI compliance.

    Payment SDKs

    The Payment SDKs wrap the Payment APIs. They include server, mobile and browser SDKs.

    Batch Payment

    Batch credit card, EFT and ACH transactions.

    Recurring Payment

    Schedule series of automatic payments.

    Address Verification System (AVS)

    The Address Verification System (AVS) is a system used to verify the address of a person claiming to own a credit card.

    Risk Thresholds

    Risk Thresholds can be used by merchants to block certain high risk transactions.
